### 4.6. Вариативная часть

В рамках вариативной части проекта наша команда выбрала тему: **Python: How to Write a Lisp Interpreter in Python**.  
Целью реализации является создание Telegram-бота под названием **MoodBuddy**, предназначенного для отслеживания настроения пользователей.  

#### Используемые библиотеки:
- **python-telegram-bot**: для взаимодействия с API Telegram.  
- **sqlite3**: для хранения данных о настроении.  
- **matplotlib**: для построения круговых диаграмм.  
- **io**: для работы с графическими буферами.  

#### Принцип работы чат-бота:

1. Пользователь запускает бота с помощью команды **/start**.  
2. Команда **/mood** инициирует запрос на запись текущего настроения:  
   - Если ввод корректен (выбрано одно из: *Отличное*, *Хорошее*, *Грустное*, *Плохое*), настроение записывается в базу данных.  
   - Если ввод некорректен, бот просит выбрать одно из предложенных настроений.  
3. При вводе команды **/stats** бот возвращает статистику за последние **7** или **30** дней в виде круговой диаграммы.  
   - Если за указанный период нет данных, бот сообщает об этом.  
4. Команда **/remind** активирует ежедневные напоминания о записи настроения в **20:00**.  
5. Пользователь получает визуализированную статистику в виде диаграммы с процентным соотношением положительных и отрицательных эмоций.  

#### Пример работы бота:
- Команда: **/mood**  
- Ответ бота: *Какое у тебя настроение? Выбери: Отличное, Хорошее, Грустное, Плохое*  
- Пользователь: *Хорошее*  
- Ответ бота: *Настроение "Хорошее" записано!*  
- Команда: **/stats 7**  
- Ответ бота: *Диаграмма с процентным соотношением за 7 дней*  
- Команда: **/remind**  
- Ответ бота: *Напоминания включены! Я буду писать каждый день в 20:00.*  

Таким образом, разработанный Telegram-бот **MoodBuddy** успешно выполняет задачи по учету настроений, визуализации данных и напоминаниям, обеспечивая удобный интерфейс и функциональность для пользователей.
